# 智能垃圾分类助手 — 零基础启动教程

> 本教程适合完全不了解后端/Docker 的同学。按顺序一步步操作即可。

---

## 项目由哪几部分组成？

```
你的电脑
├── Docker Desktop（帮你管理后端服务）
│   ├── MySQL 数据库   ← 存储垃圾分类数据
│   ├── Redis 缓存     ← 加速查询
│   ├── Flask 后端     ← AI 识别 / API 接口
│   └── Nginx 反向代理 ← 统一对外暴露端口 80
│
└── 微信开发者工具（小程序前端）
    └── 连接到 http://localhost（即上面 Nginx 的 80 端口）
```

**你每次启动只需要做两件事：**
1. 启动 Docker Desktop → 运行 `docker-compose up -d`
2. 打开微信开发者工具 → 点编译

## 第一步：启动 Docker Desktop

1. 在电脑任务栏右下角找到 **Docker** 图标（鲸鱼形状🐳）
2. 双击打开，等待图标变为**绿色**（不再旋转），代表 Docker 已就绪

> 如果找不到图标，从开始菜单搜索 **Docker Desktop** 打开

**判断是否就绪：**
打开任意终端（Win + R → 输入 `cmd` → 回车），输入：
```
docker ps
```
如果看到一行表头（即使没有容器在跑），说明 Docker 正常。

## 第三步：一键启动所有后端服务

在终端输入：

```powershell

```

**第一次会比较慢**（需要下载镜像），之后再启动就很快了。

### 如何确认都启动成功了？

**方式一：命令行查看**
```powershell
docker-compose ps
```
四个服务的 STATUS 都应该显示 `running`。

**方式二：浏览器访问**
打开浏览器，访问：
```
http://localhost/api/v1/health
```
看到如下内容说明后端正常：
```json
{"code": 200, "message": "success", "data": {"status": "ok"}}
```

## 常见问题排查

### 问题一：`docker ps` 提示"daemon is not running"

**原因：** Docker Desktop 没有启动。

**解决：** 打开 Docker Desktop，等鲸鱼图标变绿，再试。

---

### 问题二：`docker-compose up -d` 后某个服务 STATUS 不是 running

**排查步骤：**
```powershell
docker-compose logs backend
```
把报错内容截图，大概率是数据库还没完全启动，等 30 秒后再执行：
```powershell
docker-compose restart backend
```

---

### 问题三：浏览器访问 `http://localhost/api/v1/health` 没有响应

**检查 Nginx 是否在跑：**
```powershell
docker-compose ps
```
如果 nginx 的 STATUS 不是 running，执行：
```powershell
docker-compose restart nginx
```

---

### 问题四：微信开发者工具提示"request:fail"

**原因：** 没有勾选"不校验合法域名"。

**解决：** 详情 → 本地设置 → 勾选该选项，重新编译。

---

### 问题五：小程序一直显示"加载中"或白屏

**检查后端是否正常运行：**
浏览器访问 `http://localhost/api/v1/health`，如果访问不通，先排查第三步。

## 端口说明（仅供参考）

| 服务 | 本机端口 | 说明 |
|------|---------|------|
| Nginx | `80` | 小程序连接此端口 |
| Flask 后端 | `5000` | 也可直接访问（绕过 Nginx） |
| MySQL | `3307` | 本机数据库工具可用此端口连接 |
| Redis | `6379` | 缓存服务 |

> 小程序配置的地址是 `http://localhost`（即 80 端口），无需改动。

## 快速命令速查表

| 操作 | 命令 |
|------|------|
| 启动所有服务 | `docker-compose up -d` |
| 查看运行状态 | `docker-compose ps` |
| 查看后端日志 | `docker-compose logs -f backend` |
| 重启某个服务 | `docker-compose restart backend` |
| 停止所有服务 | `docker-compose down` |
| 强制重建镜像 | `docker-compose up -d --build` |

> 所有命令都需要在 `garbage-classifier/` 目录下执行。
